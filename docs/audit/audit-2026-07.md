# WaaS Audit — full report (2026-07-08)

Component-by-component audit (`operator/`, `api-server/`, `frontend/`,
`wwt/`, `hack/`), no implementation. Every finding is sourced (file +
line); coverage figures are measured, not estimated. Scope corrections
relative to the brief: the operator is **controller-runtime Go** (not
Kopf), the backend is called `api-server/`, and the canonical CI is
**GitHub Actions** (GitLab in transition, removal planned).

## 1. Mapping

| Component | Real role | Boundaries / dependencies |

## 2. Duplicates — with the divergences that have already bitten

### D1 (the most important) — the protocol list exists in 4 copies

- `operator/api/v1alpha1/workspacetemplate_types.go:207` (kubebuilder enum)
- `operator/api/v1alpha1/workspaceimage_types.go:9` (kubebuilder enum, 2nd copy)
- `operator/pkg/params/params.go:441` (`Protocols()`)
- `api-server/internal/service/governance_service.go:331` (`AdminUpsertImage` switch)

**Proof of real-world harm**: adding kasmvnc required 4 manual edits,
and the switch (4th copy) nearly got forgotten. No exhaustiveness test
ties these 4 sources together — unlike the overridable-fields registry
which has exactly that safeguard
(`operator/pkg/policy/overrides_registry_test.go`, a model to
replicate). *Factoring: `Protocols()` = single source + a "CRD enum ==
Protocols()" test (YAML scan, as the overridable-fields test already
does) + switch replaced by a lookup.*

### D2 — Hand-written TS types vs Go models

`frontend/src/types.ts` (642 lines) copies
`api-server/internal/model/model.go`. Divergence already paid for: the
`params: null` regression (the TS type lied about nullability).
Latent lies of the same kind: any Go slice/map field **without**
`omitempty` serializes as `null` where TS declares an array.
*Factoring: generation (tygo, or OpenAPI + openapi-typescript —
`docs/openapi-governance.yaml` exists but is manual and partial, so is
itself a drifting duplicate).*

### D3 — Duplicated phases

Go enum (`operator/api/v1alpha1/workspace_types.go:13-25`) vs
scattered TS literals (`frontend/src/lib/lifecycle.ts:18-23`,
`frontend/src/pages/PortalPage.tsx:448,493,496`,
`frontend/src/lib/target.ts`). No incident yet; a phase renamed on the
Go side would silently break the cards. *Single TS union type in
types.ts (or D2 solves it).*

### D4 — Copy-secret-to-pod-namespace ×2

`operator/internal/controller/kasm_credentials.go` (~60 lines,
per-workspace, owner-ref/teardown) and
`operator/internal/controller/pull_secret.go` (~70 lines, shared,
dedicated label) both implement "ensure a Secret in the target ns with
convergence". Deliberately different semantics but an identical
get/create/update-on-drift skeleton. *Possible factoring into an
`ensureSecretCopy(key, data, labels, owner)` helper — modest gain, do
it at the 3rd occurrence.*

### D5 — micro-duplicates (cosmetic)

`envOr` ×2 (`wwt/cmd/main.go:61`,
`api-server/internal/config/config.go:182`); the CSS constant
`field = 'mt-1 w-full rounded-md…'` repeated in ≥4 dialogs;
`eventLastSeen` (the 3 Event timestamps) will stay unique as long as
no 2nd consumer appears.

**Semantic duplication that's monitored but HEALTHY**: the rule
"what the portal displays = what the webhook decides" is NOT
duplicated — both call `pkg/policy` (the repo's best architecture
decision, to be protected).

## 3. Standards by component

### Operator — broadly compliant, 3 significant gaps

✅ Compliant (verified): status subresource + `metav1.Condition` +
`observedGeneration`, printcolumns, events on transitions, idempotent
finalizer, RBAC enumerated **with no wildcard** + `rbac_test.go`
safeguard (generated ⊆ helm), leader election + probes wired in the
chart (`helm/waas/templates/operator.yaml:150,178`), Secret/ConfigMap
reads outside the cache, no in-memory state between reconciles
(`Now`/`Probe` injectable = configuration, not state).

| Severity | Finding |
|---|---|
| **Important** | **No envtest**: 70.6% controller coverage but 100% fake-client. CRD defaulting, **CEL** (the image/registry one-of!), SSA/conflicts, real GC are only exercised by the k3d smoke test. The fake client has already masked a real bug (pvc-protection vs cleanupNamespace, cf. `docs/workspace-deletion.md`) |
| **Important** | `ensureDeployment/STS` **create-only on the podTemplate** (`operator/internal/controller/workload.go:205+`): an edited template (image, env, resources) never converges to existing workloads — except the kasmvnc hash, the only converged field. An implicit choice that has become a rule: to be **documented as a decision** (or converged field by field) — this is an admin's #1 surprise |
| **Important** | `v1alpha1` CRDs with no written evolution strategy (no conversion/v1beta1 plan, fields added continuously). Fine today, to be formalized before any external user |
| Cosmetic | `app.kubernetes.io/name`/`instance` labels absent from objects created by the operator (only `managed-by` + `waas.xorhub.io/*`); `workspace_controller.go` = 921 lines (lifecycle/status/finalize split possible) |

### api-server (Go) — idiomatic, 2 significant gaps

✅: errors consistently wrapped, `context` propagated, uniform
problem+json, real server-side input validation, 100% server-side
authorization (`fetchByID`, actor), secrets never logged (verified on
credential paths), goroutines with ctx (sweepers, watches), golangci
v2 configured, constructor-based DI, testable.

| Severity | Finding |
|---|---|
| **Important** | `http.Server` with **only `ReadHeaderTimeout`** (`api-server/cmd/api-server/main.go:143-146`): no `ReadTimeout`/`IdleTimeout`. Nuance: `WriteTimeout` MUST STAY 0 (it would kill SSE) — but that deserves a comment, and the other two are missing |
| **Important** | **Dual-backend** SQL layer (pgx prod / sqlite dev) at **15.5%** coverage, 2 test functions: the subtleties already encountered (RFC3339 scanners, `repository/time.go`) prove this is a divergence-bug-prone zone |
| Cosmetic | `governance_service.go` 811 lines / `workspace_service.go` 914 lines — catch-all services, splittable |

### Frontend — excellent hygiene, near-absent safety net

✅: `strict: true` effective, **0 `any`**, **0 `console.log`**,
react-query consistent everywhere (a single server-state pattern),
loading/error/empty handled in recent components, no client-side
authorization logic (everything is re-fetch-authorized).

| Severity | Finding |
|---|---|
| **Important** | **No eslint/prettier at all** (`tsc -b` is the only lint). No react-hooks/exhaustive-deps rules: `DesktopPane.tsx`'s effects (l. 144-299) rely on manual discipline |
| **Important** | Real coverage is **7.9%** of statements (155/1959 — the "naive" 36% only counts files imported by tests). All page logic (PortalPage **1617 lines**, TemplatesPage 753 lines) is untested |
| **Important** | Nullability: the `?? []` contract is applied case by case; without D2 (type generation), every new Go field re-creates the `params: null` risk |
| Cosmetic | PortalPage monolith (dialogs to extract); basic a11y unaudited (icon buttons with no aria-label in SessionCard/menus) |

### wwt — good shape

✅: 87 / 86.8 / 62.2% coverage on guac/kasm/proxy, documented and
tested security invariant (token validated before any dial),
`ReadHeaderTimeout` present, no goroutine leaks (double-done pipe).
Cosmetic gaps: `envOr` duplicated; TLS mode (`WWT_TLS_*`) is exercised
nowhere.

### hack — healthy

✅: `set -eu` everywhere (POSIX sh, pipefail not applicable),
`audit-orphans.sh` documented at the top and driven by the inventory
(no hardcoded list), seeds guarded by the **real webhook** in test
(`seed_manifests_test.go`) — an exemplary pattern. Nothing to promote
into code: these scripts are at the right level. `crd_to_jsonschema.py`
has no test but is indirectly covered by the kubeconform job.

## 5. Technical debt & prioritized action plan

**Inventoried debt** (the repo has no TODOs — the debt lives in the
docs, which is better than average):

- Documented but unfixed N+1 (`docs/diagnostics/workspace-list-loading.md`
  — per-workspace template Get × 15s quota poll; the fix is already
  written in the diagnostic).
- Ungoverned clipboard on the kasmvnc path (v1 decision, accepted as
  is, kasm phase 4 = DLP derived from the policy).
- `guacamole-common-js ^1.5` vs guacd 1.6.0 (`frontend/package.json:15`
  vs `helm/waas/values.yaml:141` — compatible but to be aligned).
- OIDC never tested against a real IdP.
- `registry.xorhub.io` references in gitops (4 occurrences).
- Local golangci toolchain pin vs CI (documented incident).
- `.gitlab/` + `cliff.toml` to remove at the end of the GitHub transition.

**Plan (quick wins → bigger efforts)**:

| Order | Effort | Problem / benefit | Risk | Size |
|---|---|---|---|---|
| 1 (QW) | Protocol exhaustiveness guard (D1) | kills the most recently experienced bug class | none | ~1h |
| 2 (QW) | eslint + prettier + react-hooks + CI job | nonexistent frontend safety net → hooks/a11y rules | low (hygiene already good) | ~½ day |
| 3 (QW) | `http.Server` timeouts + WriteTimeout/SSE comment | network robustness | none | ~30 min |
| 4 | **TS type generation** (D2): tygo on `model.go` → `types.gen.ts`, page-by-page migration | permanently kills D2, D3, and the `params: null` bug class — best ratio in the plan | medium (progressive migration) | ~2 days |
| 5 | Handler + dual-backend repository tests (#1, #2) | riskiest backend zones | low | ~2-3 days |
| 6 | Targeted envtest (#3) | prerequisite for any confident CRD evolution | low | ~2 days |
| 7 | PortalPage split + component tests (#4) | 1617-line monolith, blind refactors | medium | ~2 days, shippable per dialog |
| 8 | Document the "create-only podTemplate" doctrine (or converge) + CRD evolution strategy | implicit structuring decision | decision to make | ~½ day |
| 9 | Fix the documented N+1 | portal perf | low | ~½ day |

**Anti-debt (preventing its return)**:

- (a) eslint + prettier CI job.
- (b) Type generation in CI with `git diff --exit-code` (same pattern
  as the existing CRD drift check — already a cultural mechanism in
  this repo).
- (c) Extend the "registry + exhaustiveness test" pattern (overridable
  fields) to protocols and phases.
- (d) Per-package coverage ratchet on the flagged zones (handler ≥ 40%,
  repository ≥ 50%).
- (e) Lightweight pre-commit gofmt + eslint (optional — CI is already
  blocking).
- (f) Documented convention + custom lint: no `json:"…"` without
  `omitempty` on slices/maps in API models (THE root cause of
  `params: null`).
